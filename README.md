# 청소난투(욕실의 난)

욕실을 테마로 한 4인 개인전 실시간 대전 게임 프로젝트.

이 저장소는 전체 시스템 중 `모바일 앱(게임 콘솔 / 러버덕 조작기)` 구현을 우선 대상으로 한다.  
웹 캔버스 게임 화면, 서버 연동, 장애물 이벤트 등 전체 구조를 함께 고려하되, 실제 구현 초점은 앱 측 입력과 상태 전달에 둔다.

## 1. 게임 개요

- 게임명: 청소난투(욕실의 난)
- 장르: 4인 실시간 개인전
- 콘셉트: 욕실 안의 유기기구/장치 특성을 활용한 난투 게임
- 플레이어 캐릭터: 러버덕
- 핵심 목표: 맵에 숨겨진 `깃발(상징물)`을 차지하고 오래 들고 있기

## 2. 승리 규칙

- 가장 중요한 점수 요소는 `깃발 보유 누적 시간`
- 최종 우승자는 깃발을 가장 오래 들고 있던 플레이어
- 2등, 3등도 깃발 보유 시간 순으로 결정
- 게임 종료 시점에 깃발을 들고 있는 플레이어는 `보너스 점수` 획득
- 플레이어끼리 부딪히면 깃발을 탈취할 수 있음

## 3. 기본 플레이 시나리오

- 총 4명이 동시에 접속
- 시작 위치는 맵의 4개 모서리
- 게임 시간은 우선 `무제한` 기준으로 검토
- 맵 어딘가에 깃발이 숨겨져 있으며, 이를 먼저 찾고 유지하는 것이 핵심
- 장애물은 MVP 테스트 후 실제 등장 타이밍과 세기를 조정

## 4. 장애물 기획

### 장애물 1. 배수구

- 맵 중앙 고정 위치
- 주기적으로 열리며 플레이어를 끌어당김
- 현재 우선순위가 가장 높은 장애물

### 장애물 2. 수도꼭지 효과

- 주기적으로 많은 물이 쏟아짐
- 물의 힘으로 러버덕을 밀어냄

### 장애물 3. 비치볼

- 통과할 수 없는 물리 장애물
- 실제 도입 여부는 MVP 테스트 후 판단

## 5. 캐릭터 및 상태

### 스킨

- 일반 러버덕
- 깃발을 든 러버덕

### 플레이어 상태값

- 현재 깃발 보유 여부
- 깃발 보유 누적 시간
- 마지막 순간에 깃발을 들고 있었는지 여부

## 6. 랭킹 및 UI 정보

- 4명의 깃발 보유 시간을 실시간으로 표시
- 가장 오래 보유한 순서대로 순위 집계
- 게임 시작 타이머 필요
- 배경음악 재생 필요
- 배경 영상/연출과 장애물 등장 타이밍을 연동할 수 있어야 함

## 7. 화면/맵 요구사항

- 메인 캔버스 기준 해상도: `1700 x 1080`
- 게임 맵 렌더링 필요
- 중앙 배수구 위치는 고정
- 장애물 등장 시간에 맞춰 캔버스에서 효과를 트리거해야 함

## 8. 시스템 구성 초안

### 캔버스

- 웹페이지 기반 게임 화면
- 게임 상태 시각화
- 장애물 연출 및 효과 처리

### 서버

- pub/sub 서버 사용
- 후보: `Azure Web PubSub`
- 플레이어 상태, 이동 이벤트, 점수/시간 집계 동기화 담당

### 모바일 앱

- 러버덕 조작용 게임 콘솔 역할
- 자이로스코프 기반 조향
- 앞/뒤/좌/우/대각선 이동 지원
- 회전 기능은 후순위 옵션
- 앱은 조작 입력 전송에 집중
- 서버와의 통신은 `MQTT` 방식 고려

## 9. 이 저장소의 1차 구현 범위

이 저장소는 우선적으로 아래 항목 구현을 목표로 한다.

- 모바일 앱 프로젝트 구조 설계
- 러버덕 이동 조작 UI/입력 체계 정의
- 자이로스코프 기반 이동 입력 처리
- 서버로 이동 이벤트 전달하는 인터페이스 설계
- 플레이어 상태 표시를 위한 최소 모델 정의

## 10. MVP 우선순위

1. 4인 접속을 가정한 기본 조작 흐름 정리
2. 앱에서 이동 입력을 생성하고 서버로 전달
3. 깃발 보유 상태 및 보유 시간 상태 모델 정의
4. 중앙 배수구 이벤트 대응을 위한 최소 게임 이벤트 구조 설계
5. 수도꼭지/비치볼 등 추가 장애물은 후속 확장

## 11. 미정 항목

- 게임 종료 조건의 구체화
- 점수 계산식 상세 정의
- 깃발 발견/생성 방식
- 장애물 주기와 강도
- 모바일 앱 기술 스택(Flutter 기준 세부 구조 포함)
- Azure Web PubSub 와 MQTT의 실제 연결 구조 및 역할 분담

## 12. 한 줄 요약

`청소난투`는 욕실 테마의 4인 러버덕 난투 게임이며, 이 저장소는 그중 모바일 앱 기반 조작/입력 시스템 구현을 위한 출발점이다.
